Proxy Design Pattern
===================
Designs patterns are best practices to solve the re-occurrence problem of your application
Proxy : proxy means dummy that looks like real. Proxy internally usese real if required.

#use cases
- On normal days flipcart has no offers on purchasing any products
  but, on sale days flipcart has amazing offers on purchasing any products.

#components of proxy design patterns
1- Component interface
2- Component concrete class 
3- proxy class - - implements component interface and has component concrete class ka object
4- Factory class - factory class will give the object of either proxy class or real class

Proxy design example : 

 | -> src
	--> com.hamza.component
	    . FlipcartService.java (i)
	    . FlipcartServiceImpl.java

	--> com.hamza.proxy
	    . FlipcartServiceProxy(c)

	--> com.hamza.factory
	    . EcommerceFactory.java

	--> com.hamza.test
	    . ProxyDPTest.java
